How the levers map to the build

  • Two new per-viewbook fields on the viewbook theme/config — e.g. revealSpeed (a multiplier, default 1.0) and firstLoadDelayMs (default 3000). Both plumb into the reveal as CSS custom properties (--speed) and a JS timer, exactly as demoed.
  • Speed scales everything at once: hero min-height, image scale(1.28→1), eyebrow/title/rule, and each body item's duration and stagger delay all multiply by --speed, so the choreography never drifts out of balance.
  • Delay is per-device, once: the timer only runs when the localStorage first-visit flag is unset (per the existing vb-presentation-mode pattern). Instant / 0s expands with no countdown.
  • Presets are just stored multipliers — operators pick Grand/Standard/Brisk/Snappy in the editor, or drop to the slider for a custom value. Same idea as the existing theme picker.
